### Rotas implementadas
| Rota | O que tem |
|---|---|
| `/` | Lobby: "Jay Academy" centralizado + botão "Acessar painel" + ícones de micropigmentação girando em 2 anéis |
| `/dashboard` | Banner WordPress + LPs agrupadas (Website/LPs/Formulários) + seção "Páginas WP" + seção "Detectadas" |
| `/lps/[slug]` | Detalhe de cada LP do portal (PMU CLASS, Magic Shadow 3, Jayo Laser) |
| `/lps/connect/[folder]` | Form pra registrar pasta detectada como LP |
| `/wordpress` | Triagem de 92 páginas WP + botão "Copiar agora" com modal de confirmação |
| `/wp-pages/[domain]/[slug]` | Detalhe de página WP copiada com 3 botões "Mover pra Website/LP/Formulário" + botão "Excluir" + link "Abrir preview em nova aba" |
| `/wp-pages/[domain]/[slug]/preview` | Route handler que serve o HTML completo da página WP em tela cheia |
| `/analytics`, `/settings` | Existem mas escondidos da sidebar (acesso só por URL direta) |

### Sidebar
- Logo "Jay Academy" no topo
- Lista de LPs **colapsável** (default fechada, clica em "Páginas" e abre)
- Botão "Sair" no rodapé

### Integração WordPress
- REST API conectada (sem auth, só leitura): `jayacademy.com.br` (63 páginas) + `lp.jayacademy.com.br` (29 páginas) = **92 total**
- `fetchPageContent()` faz 2 requests: REST API pra metadados + HTML público completo (`fullHtml`) com CSS+imagens
- Triagem com 3 botões por linha: 🟢 Copiar · ⚪ Ignorar · 🔴 Desmarcar
- Botão "Marcar sugeridas" aplica heurística automática (campanhas/versões antigas → ignore, resto → copy)
- Botão "Desmarcar todas" reseta tudo
- Decisões salvas em `data/wp-decisions.json` (persistente)
- **Botão "Copiar agora"** com modal inteligente:
  - Se nada copiado antes → copia direto
  - Se algumas já copiadas → modal com 3 opções: "Copiar só as novas (X)" / "Recopiar todas (sobrescreve)" / "Cancelar"
- Páginas copiadas salvas em `data/wp-content/[domain]_[slug].json` (com `content` do editor + `fullHtml` da página pública)
- Badge "✓ COPIADA" nas linhas individuais quando já foi baixada
- Seção "Já copiadas pro portal" colapsável na página /wordpress
- Categorização visual: páginas separadas em 2 blocos — "Páginas" (não-campanhas) e "Campanhas e ações"

### Sistema de categorização (no dashboard)

## ⚠️ Notas importantes

- **Hot reload**: às vezes `revalidatePath` não atualiza o browser. Solução: F5 manual.
- **IDE diagnostics**: Next.js às vezes deixa cache stale do TypeScript. Diagnostics tipo "Cannot find name 'X'" depois de Edit podem ser cache — arquivo na verdade está correto. Verificar com Read antes de mexer.
- **REST API WordPress**: aceita até `per_page=100` por default. Os 2 domínios juntos têm 92 páginas, então cabe em 1 request por domínio.
- **HTML completo das páginas WP**: armazena na key `fullHtml` do JSON. Se vazio, fallback é o `content` do editor (sem CSS).
- **Triagem das 92 páginas**: a heurística automática trata bem campanhas/versões antigas mas não é perfeita. James pode ajustar manualmente.
